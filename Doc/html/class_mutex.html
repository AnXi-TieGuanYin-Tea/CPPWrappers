int Mutex::destroy ( )

The destroy() method frees the resources allocated for mutex.

Returns:
If successful, destroy() will return zero. Otherwise, an error number will be returned to indicate the error.
int Mutex::lock ( )

locks the mutex.

If the mutex is already locked, the calling thread will block until the mutex becomes available.

Returns:
If successful, lock() will return zero. Otherwise, an error number will be returned to indicate the error.
int Mutex::tryLock ( )

The trylock() method locks mutex.

If the mutex is already locked, trylock() will not block waiting for the mutex, but will return an error condition

Returns:
If successful, trylock() will return zero. Otherwise, an error number will be returned to indicate the error.
int Mutex::unlock ( )

If the current thread holds the lock on mutex, then the unlock() function unlocks mutex.

Calling unlock() with a mutex that the calling thread does not hold will result in undefined behavior.

Returns:
If successful, unlock() will return zero. Otherwise, an error number will be returned to indicate the error.
